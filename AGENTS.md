## Repo Structure

```
├── hebrew_dates_to_cal/   # Python CLI — CSV → ICS converter
│   ├── parser.py          # Hebrew text → HebrewDateSpec (gematria, month aliases)
│   ├── scheduler.py       # HebrewDateSpec → Gregorian dates (pyluach)
│   ├── models.py          # Frozen dataclasses + title rendering
│   ├── ics.py             # RFC 5545 ICS builder
│   └── cli.py             # Argparse entry point
├── app/                   # React Native (Expo) — Android/iOS app
│   ├── src/core/          # TS port of Python logic
│   │   ├── types.ts       # HebrewDateSpec, ScheduledEvent interfaces
│   │   ├── hebrewDate.ts  # Gematria, month aliases, picker data
│   │   ├── scheduler.ts   # Hebrew → Gregorian via @hebcal/core
│   │   ├── ics.ts         # ICS builder (ported from Python)
│   │   └── index.ts       # Barrel exports
│   ├── src/components/    # HebrewDayPicker, HebrewMonthPicker, YearsPicker
│   ├── src/screens/       # MainScreen (single-screen app)
│   ├── src/__tests__/     # Jest tests for core logic
│   └── android/           # Generated by expo prebuild (not committed)
├── Makefile               # install, test, build-apk, clean
└── output/                # Built artifacts (hebrew-dates.apk)
```

## Commands

```bash
# Python CLI
.venv/bin/pip install -e .
.venv/bin/python -m hebrew_dates_to_cal data/family_dates.csv output/out.ics --occurrences 120
.venv/bin/python -m pytest tests/

# App
cd app && npm install && npm test
make build-apk              # prebuild + gradle → output/hebrew-dates.apk
adb install -r output/hebrew-dates.apk
```

## Code Style

- Pure functions, pipeline pattern, single responsibility, type hints everywhere
- No try/except on imports — deps in pyproject.toml / package.json
- UX: fewest steps, sensible defaults, obvious thing just works

## Key Tech Decisions

- **Expo** (not bare RN): easier prebuild, managed native modules
- **@hebcal/core**: Hebrew↔Gregorian (ESM-only)
- **expo-intent-launcher**: opens ICS directly in Google Calendar (not expo-calendar)
- **No I18nManager.forceRTL**: explicit RTL styles instead

## Lessons & Gotchas

- `I18nManager.forceRTL(true)` double-flips `textAlign:"right"` → appears left. Use explicit `flexDirection:"row-reverse"` + `textAlign:"right"` without forceRTL.
- `@hebcal/core` is ESM-only. Jest needs `moduleNameMapper` to `dist/esm/index.js` + `transformIgnorePatterns`.
- `jest-expo` conflicts with Jest 30. Use `ts-jest/presets/default-esm` for pure TS tests.
- `expo prebuild --clean` wipes `android/local.properties`. Re-create with `sdk.dir=` before Gradle.
- `expo-sharing` opens generic share sheet. Use `expo-intent-launcher` + `ACTION_VIEW` + `text/calendar` to open in Calendar.
- `FileSystem.getContentUriAsync()` converts `file://` → `content://` URI for Android intents.
